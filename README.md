# Workday Insight

เว็บแอป Google Apps Script สำหรับสรุปเวลาสายและวันลา โดยเก็บข้อมูลใน Google Sheets

## ความสามารถ

- เลือกช่วงวันที่เริ่มต้น–สิ้นสุด และกรองตามพนักงาน
- เพิ่ม แก้ไข และลบข้อมูลพนักงาน ยอดรวมเวลาสายรายเดือน และวันลา
- แสดงช่วงค้นหาเริ่มต้นเป็นวันแรกถึงวันสุดท้ายของเดือนปัจจุบัน
- ลบพนักงานพร้อมรายการที่เกี่ยวข้องหลังผู้ใช้ยืนยัน
- คำนวณสิทธิ์ลาคงเหลือตามปีของวันสิ้นสุดที่เลือก
- วิเคราะห์ข้อมูลจริงด้วย Gemini 2.5 Flash
- ส่งออกสรุปพนักงานเป็น CSV

## ติดตั้งใน Google Sheets

1. เปิด Google Sheet ที่ใช้เป็นฐานข้อมูล แล้วเลือก **Extensions → Apps Script**
2. สร้างหรืออัปเดตไฟล์ Code.gs, index.html, Styles.html, Scripts.html และ appsscript.json จากโปรเจกต์นี้
3. ตั้งค่า Time zone เป็น Asia/Bangkok
4. รันฟังก์ชัน setupDatabase หนึ่งครั้งและอนุญาตสิทธิ์ ระบบจะสร้างชีต employees, attendance และ leave
5. Deploy → New deployment → Web app → Execute as: Me → เลือกสิทธิ์การเข้าถึงที่เหมาะกับองค์กร
6. เมื่อแก้โค้ดภายหลัง ให้สร้าง deployment version ใหม่

ข้อมูลเดิมที่มีคอลัมน์แบบเก่าจะถูกย้ายอัตโนมัติ โดยเพิ่ม Record ID ให้รายการเวลาสายและวันลา เพื่อรองรับการแก้ไขและลบ

เวลาสายบันทึกเป็นยอดรวมรายพนักงานต่อเดือน โดยใช้วันที่ 1 ของเดือนเก็บในชีต `attendance` เพื่อให้ยังใช้งานกับโครงสร้างข้อมูลเดิมได้ หากบันทึกพนักงานและเดือนเดิม ระบบจะอัปเดตยอดเดิมและรวมรายการซ้ำของเดือนนั้นให้เหลือรายการเดียว การกรองเวลาสายจะพิจารณาตามเดือนที่ครอบคลุมในช่วงวันที่ ส่วนวันลายังกรองตามวันที่จริง

## เปิดใช้งาน AI

1. สร้าง Gemini API key ใน Google AI Studio
2. ใน Apps Script เปิด **Project Settings → Script Properties**
3. เพิ่ม Property ชื่อ GEMINI_API_KEY และวาง API key เป็นค่า
4. Deploy เวอร์ชันใหม่ แล้วเปิดหน้า **AI Insight**

ระบบจะส่งเฉพาะข้อมูลสรุปในช่วงเวลาและพนักงานที่ผู้ใช้เลือกไปยัง Gemini API ควรกำหนดสิทธิ์เข้าถึง Web app ให้เหมาะกับข้อมูลบุคลากรและนโยบายขององค์กร

## สิทธิ์ลาเริ่มต้น

- ลาป่วย 30 วันต่อปี
- ลากิจ 5 วันต่อปี
- พักร้อนคำนวณจากอายุงาน ณ วันที่ 1 มกราคม: ต่ำกว่า 1 ปี = 0 วัน, 1–2 ปี = 6 วัน, 3–5 ปี = 8 วัน, 6–8 ปี = 10 วัน, 9–11 ปี = 12 วัน, 12–14 ปี = 14 วัน และ 15 ปีขึ้นไป = 15 วัน
